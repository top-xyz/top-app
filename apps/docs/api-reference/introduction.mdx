# API Reference

Welcome to the Top API reference. Our API follows REST principles and uses standard HTTP methods and status codes.

## Authentication

All API requests require authentication using a bearer token:

```bash
curl -H "Authorization: Bearer YOUR_API_KEY" \
  https://api.top.dev/v1/contexts
```

You can obtain an API key from your [Top Dashboard](https://app.top.dev/settings/api).

## Base URL

```
https://api.top.dev/v1
```

## Rate Limiting

- 1000 requests per minute per API key
- Rate limit headers included in responses
- Exponential backoff recommended

## Response Format

All responses are in JSON format and include:

```json
{
  "data": {
    // Response data
  },
  "meta": {
    "requestId": "req_123",
    "timestamp": "2024-01-15T12:00:00Z"
  }
}
```

## Error Handling

Errors follow standard HTTP status codes:

```json
{
  "error": {
    "code": "invalid_request",
    "message": "Invalid request parameters",
    "details": {
      "field": "contextId",
      "issue": "Required field missing"
    }
  }
}
```

Common status codes:
- `400` Bad Request
- `401` Unauthorized
- `403` Forbidden
- `404` Not Found
- `429` Too Many Requests
- `500` Internal Server Error
